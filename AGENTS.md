# Web Cam PWA Agent Guide

## 목적

- 이 저장소의 1차 목적은 **PWA 기술 이해**입니다.
- 완성도 높은 제품 구현보다, 핵심 개념을 단계적으로 학습하는 것을 우선합니다.

## 커뮤니케이션 원칙

- 답변은 한국어로 작성합니다.
- 한 번에 크게 구현하지 않고, **짧은 텀의 대화**로 진행합니다.
- 구현 전에 현재 단계의 목표와 작업 내용을 짧게 합의합니다.
- 사용자가 요청하지 않은 대규모 리팩터링/기능 확장은 하지 않습니다.

## 기술 선택 원칙

- 초기 구현은 **Vanilla HTML/CSS/JavaScript**를 사용합니다.
- React 등 프레임워크는 학습 목적상 초기 단계에서 사용하지 않습니다.
- PWA 필수 요소를 명시적으로 다룹니다:
  - `manifest.webmanifest`
  - `service worker`
  - `HTTPS` 또는 `localhost` 환경
  - `getUserMedia` 기반 카메라 권한/접근 처리

## 개발 단계(권장 순서)

1. 카메라 미리보기(`<video>`) 동작
2. 사진 캡처(`<canvas>`) 및 결과 표시
3. PWA 기본 구성(manifest, service worker, 앱 셸 캐시)
4. 오프라인/설치 가능 여부 점검
5. 필요 시 로컬 저장(IndexedDB 등) 확장

## 브라우저/테스트 가이드

- Chrome을 우선 검증 대상으로 삼되, Chrome 전용으로 가정하지 않습니다.
- 브라우저별 지원 차이(설치 UX, 권한, SW 동작)는 필요 시 명시합니다.
- 단계별 검증 시 Chrome DevTools(Application/Service Worker/Manifest) 확인을 권장합니다.

## 코드 작성 원칙

- 각 단계에서 최소 동작 예제를 먼저 만듭니다.
- 복잡한 추상화보다 읽기 쉬운 구조를 우선합니다.
- 예외/권한 거부 케이스를 기본 처리합니다.
